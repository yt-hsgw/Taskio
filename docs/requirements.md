# Taskio Requirements（Draft）

## 1. 概要（Overview）

**Taskio** は、個人のタスク実行を「記録可能なログ」として管理することを目的としたタスク管理アプリケーションである。

一般的な Todo アプリとは異なり、Taskio では以下を重視する：

- タスクの「開始」「終了」を明示的に記録する
- タスク実行中の状態をログとして残す
- クライアント（Android / Web）とサーバーを分離した API 駆動設計

本プロジェクトは **個人開発** を前提とし、将来的な機能拡張（永続化、認証、分析）を見据えた構成とする。

---

## 2. 目的（Goals）

- タスク実行の事実を正確に記録できる
- 実装を通じて Server / Client 分離アーキテクチャを学習する
- Android クライアントを主軸とし、Web クライアントにも展開可能な API を提供する

---

## 3. スコープ（Scope）

### 3.1 対象範囲（In Scope）

- タスクの CRUD
- タスクログの CRUD
- タスク開始・終了の記録
- REST API（JSON）
- Android クライアントからの疎通確認

### 3.2 対象外（Out of Scope）※現時点

- ユーザー認証・認可
- マルチユーザー対応
- 永続化（DB）
- 集計・分析機能

---

## 4. 用語定義（Glossary）

| 用語 | 説明 |
|---|---|
| Task | ユーザーが実行する作業単位 |
| TaskLog | Task の実行ログ（開始・終了など） |
| Client | Android / Web アプリケーション |
| Server | Rust + Axum による API サーバー |

---

## 5. 機能要件（Functional Requirements）

### 5.1 タスク管理

- タスクを作成できる
- タスク一覧を取得できる
- タスク詳細を取得できる
- タスクを更新できる
- タスクを削除できる

#### タスク属性

- ID（UUID）
- タイトル（必須）
- 説明（任意）

---

### 5.2 タスクログ管理

- タスクに紐づくログを作成できる
- タスクに紐づくログ一覧を取得できる
- ログ詳細を取得できる
- ログを更新できる
- ログを削除できる

#### タスクログ属性

- ID（UUID）
- task_id（紐づく Task）
- 開始時刻
- 終了時刻（任意）
- メモ（任意）

---

### 5.3 タスク実行フロー

- ユーザーは明示的な操作（ボタン押下）でタスクを開始する
- 開始時に TaskLog が作成される
- 終了操作により TaskLog が更新される

---

## 6. 非機能要件（Non-Functional Requirements）

### 6.1 API

- RESTful API
- JSON フォーマット
- バージョニング（/api/v1）

### 6.2 クライアント

- Android（Jetpack Compose）
- Retrofit + Moshi

### 6.3 サーバー

- Rust
- Axum
- In-memory データ管理（初期段階）

---

## 7. 制約条件（Constraints）

- 個人開発規模
- 初期は学習目的を優先
- 仕様は段階的に更新される前提とする

---
